# 验收用例运行器设计

日期：2026-06-12

## 目标

老师提供的 `作业验收用例.xlsx` 是一组带模拟时间的验收事件。系统需要能够从 06:00 按事件顺序运行到 09:30，输出每个事件后的快充桩、慢充桩和等候区状态，格式接近 Excel 中的表格。

本功能不替代现有车主端、运营管理端和调度控制台。它作为独立的“测试用例”入口，服务于验收和结果核对，避免把模拟时钟、批量事件和 Excel 表格格式污染到日常演示流程中。

## 事件解释

事件格式为 `(来源,对象,操作,数值)`。

- `(A,Vn,F,电量)`：车辆提交快充请求。
- `(A,Vn,T,电量)`：车辆提交慢充请求。
- `(A,Vn,O,0)`：车辆取消当前请求。若车辆在等候区则直接移除；若在充电区队列或正在充电，则移除并释放队列位置。
- `(B,F1,O,0)` / `(B,T1,O,0)`：对应充电桩故障。
- `(B,F1,O,1)` / `(B,T1,O,1)`：对应充电桩恢复。
- `(C,Vn,O,电量)`：车辆修改请求电量。按照需求文档，只有等候区车辆允许修改；若车辆已在充电区，则记录为不可执行事件并保持状态不变。

`C` 事件按修改电量处理，而不是结束充电。原因是老师样例中的 `C,V21,O,35` 出现在 V21 仍可能位于等候区的阶段，并且详细需求明确把修改请求作为验收功能；结束充电在普通系统演示里已有，但这张表的桩状态列更关注持续充电量和当前费用。

## 调度规则

验收运行器使用独立的内存模型：

- 快充桩 2 个，慢充桩 3 个。
- 每个充电桩队列长度 3。
- 等候区容量 10。
- 快充功率 30 度/小时，慢充功率 10 度/小时。
- 充电费按系统现有峰平谷规则计算，服务费为 0.8 元/度。

普通调度：

1. 新请求先进入等候区。
2. 若同模式充电桩队列有空位，系统自动叫号。
3. 叫号对象是等候区中同模式最早到达的车辆。
4. 目标充电桩为预计完成时间最短的同模式可用桩。
5. 进入空队列首位的车辆从当前事件时刻开始充电。

完成处理：

- 每次处理新事件前，先推进模拟时钟并结算截至当前时刻已经充满请求电量的车辆。
- 首位车辆完成后移出队列，后续车辆从完成时刻开始充电。
- 输出快照时，首位车辆显示当前已充电量和当前费用；未开始充电的队列车辆显示 0.00 和 0.00。

故障处理：

- 故障桩立即停止服务。
- 故障桩中正在充电的车辆保留已充电量，剩余电量继续作为待调度请求。
- 故障桩队列里的车辆标记为故障受影响车辆。
- 默认采用优先级调度：暂停等候区普通叫号，优先把故障受影响车辆调入其它同类型桩，直到无法继续调度或故障队列清空，然后再恢复等候区调度。

恢复处理：

- 故障桩恢复为可用。
- 将同类型未开始充电车辆按到达顺序重新整理到可用桩队列。
- 之后继续普通调度。

## 输出格式

后端返回结构化快照，并额外返回 Excel 友好的文本：

- 充电桩单元格：`(V1,0.83,1.00)`。
- 多个桩队列位置占 3 行，对应该充电桩队列长度。
- 等候区单元格：`(V13,F,110.00)-(V14,F,95.00)`。

前端展示：

- 顶部显示用例参数和运行范围。
- 中部显示事件表。
- 右侧或下方显示与老师示例行的匹配结果。
- 提供“运行用例”“复制结果”按钮。

## 非目标

第一版不直接修改老师的 Excel 文件，不实现文件上传解析，也不把验收事件写入 H2 数据库。后续如需要，可在当前结构化结果基础上追加导出 CSV 或回填 xlsx。

## 测试策略

后端测试优先：

- 06:00 第一辆慢充车进入 T1，已充电量和费用为 0。
- 06:05 V1 已充约 0.83 度，费用约 1.00，V2 进入 T2。
- 07:05 快充桩队列满后 V13 留在等候区。
- 07:10 V13 和 V14 按到达顺序留在等候区。
- T1 故障后受影响车辆优先迁移，等候区不抢占故障车辆空位。
- 09:25 对等候区车辆 V21 修改电量为 35。

前端测试优先：

- API 客户端能调用验收运行接口。
- 页面能渲染至少 36 个事件行。
- 示例匹配状态能清楚显示通过或失败。

## 风险

老师没有在 Excel 中明确定义 A/B/C 字母含义。本设计用详细需求、事件时序和样例行反推。若后续老师解释 `C` 为结束充电，只需调整事件解释和一组服务测试，不影响页面和总体回放结构。
