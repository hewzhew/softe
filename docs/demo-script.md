# 课程汇报与系统演示脚本

## 启动

1. 启动后端：

```powershell
cd D:\softe\backend
mvn spring-boot:run
```

2. 启动前端：

```powershell
cd D:\softe\frontend
npm run dev
```

3. 打开：

```text
http://127.0.0.1:5173/#/station
```

## 演示一：站点运行控制台

1. 进入“站点运行”，说明系统维护的是统一站点时钟和事件流；手工录入、预置序列、车主提交都进入同一套运行状态。
2. 点击“同步”，导入预置事件流。导入后站点时间回到 `2026-06-01 06:00:00`，事件流显示待处理和已生效事件。
3. 观察沙盘：车辆会按站点时间进入等候区、桩内队列和充电桩；事件表中的“已生效/待生效”与沙盘状态保持一致。
4. 点击“推进 5 分钟”或“推进 30 分钟”，说明系统会按时钟推进申请、调度、开始充电、充电完成、出账和下一辆车进桩。
5. 设置 `10x` 或 `60x`，点击“启动时钟”，让站点时间自动流动；需要讲解时再点击“暂停时钟”。
6. 在“事件接入”中保留事件时间为空，输入车辆编号和申请电量，点击“加入事件流”，说明人工临时订单也走同一条事件通道。

## 演示二：车主自助门户

1. 进入“车主自助”。
2. 输入车辆编号，例如 `CAR-OWNER-1`，点击“进入车辆”。如果车辆未登记，系统会为本机演示创建车辆档案；如果已登记，则直接进入门户。
3. 查看“我的车辆”，确认车辆信息独立显示，不再和申请表单混在一起。
4. 在“充电申请”选择快充或慢充，填写申请电量，点击“提交申请”。
5. 进入“当前状态”后查看排队号、前方车辆和分配桩位。
6. 返回“站点运行”点击“执行调度”或推进站点时间，再回到车主端刷新状态。
7. 车辆进入充电后，等待站点时间推进到完成，查看“账单记录”和详单。

## 演示三：运营管理工作台

1. 进入“运营管理”，查看充电桩、等候区、桩内队列和故障状态。
2. 点击“执行调度”，说明管理员仍然可以手动触发调度；站点运行台也会同步刷新。
3. 选择一个正在服务或有队列的充电桩，上报故障。
4. 选择“优先保障中断车辆”或“按请求时间重排”，确认故障后队列重新分配。
5. 点击“恢复服务”，再返回“站点运行”查看沙盘和事件状态。

## 讲解要点

- 本系统不是把测试数据放到孤立页面里播放，而是把预置序列作为事件来源接入统一站点运行台。
- 站点时钟可以暂停、倍速、手动推进；充电进度、完成结算和下一辆车进桩都由后端运行时推进。
- 车主端、运营端和站点运行台看到的是同一套本机业务数据。
- 快充请求只进入快充桩，慢充请求只进入慢充桩；普通调度目标是预计完成时间最短。
- 故障时会暂停受影响桩并按策略重排同类型车辆。
- H2 只用于降低本机部署复杂度，后续可以替换为 MySQL 等数据库。
