# 课程汇报与系统演示脚本

## 启动

1. 启动后端：

```powershell
cd D:\softe\backend
mvn spring-boot:run
```

2. 启动前端：

```powershell
cd D:\softe\frontend
npm run dev
```

3. 打开：

```text
http://127.0.0.1:5173
```

## 演示一：调度沙盘总体回放

1. 进入“调度沙盘”。
2. 加载课程事件序列。
3. 点击播放，观察车辆依次进入等候区、快充队列和慢充队列。
4. 使用“下一步”推进关键事件，说明每次请求提交、队列分配、故障和恢复后的状态变化。
5. 点击时间轴中故障前后的节点，对比受影响车辆的排队位置。
6. 播放结束后展开结果核对，确认关键时刻的车辆位置和队列状态。
7. 点击“复制结果”，将结果表复制到汇报材料或临时表格中。

## 演示二：车主自助流程

1. 进入“车主自助”。
2. 填写车辆：
   - 车辆编号：`CAR-DEMO-1`
   - 车主姓名：`Demo User`
   - 电池容量：80
   - 密码：`123456`
3. 点击“注册车辆”。
4. 点击“设置/更新密码”。
5. 选择快充，申请电量 30。
6. 点击“提交充电请求”。
7. 点击“刷新车辆状态”，查看车辆当前请求状态和队列信息。
8. 进入“运营管理”并点击刷新，查看等候区中的车辆编号。
9. 如果车辆状态提示“可开始充电”，回到“车主自助”点击“启动充电”。
10. 实际电量填 30，点击“结束充电并生成账单”。
11. 点击“查询今日账单”，查看账单；单击账单行或“查看详单”查看分时计费详单。

## 演示三：运营管理故障处理

1. 进入“运营管理”。
2. 点击刷新，查看充电桩状态、等候车辆和各桩队列。
3. 如果现场已有排队车辆的充电桩，选择对应桩，例如 `F-1`。
4. 点击“上报故障”。
5. 策略选择“优先保障中断车辆”。
6. 点击“确认上报并重调度”。
7. 查看队列状态，说明故障桩车辆优先迁移到其它同类型充电桩。
8. 对另一个故障场景选择“按请求时间重排”，说明系统按请求时间重新排列同类型未开始充电车辆。
9. 点击“恢复服务”，确认桩状态回到可服务状态。

## 可讲解要点

- 快充请求只进入快充桩，慢充请求只进入慢充桩。
- 普通排队目标是完成时间最短。
- 结束充电时系统按峰/平/谷分时电价生成账单和详单。
- 故障时暂停等候区叫号，先处理受影响车辆。
- 沙盘回放使用独立事件序列和回放状态，不污染正常业务数据。
- H2 数据库降低本机部署和课堂展示复杂度。
