# 验收演示脚本

## 启动

1. 启动后端：

```powershell
cd D:\softe\backend
mvn spring-boot:run
```

2. 启动前端：

```powershell
cd D:\softe\frontend
npm run dev
```

3. 打开：

```text
http://127.0.0.1:5173
```

## 演示一：基础调度

1. 进入“调度控制台”。
2. 设置参数：
   - 快充桩数量：2
   - 慢充桩数量：3
   - 等候区容量：10
   - 桩队列长度：2
   - 快充功率：30
   - 慢充功率：10
3. 点击“应用参数”。
4. 点击“导入排队请求”。
5. 查看充电桩表和队列表。
6. 点击“执行调度”。
7. 切换“充电区队列”，说明系统按快充/慢充模式分配到对应充电桩，并选择预计完成时间较短的队列。

## 演示二：车主完整流程

1. 进入“车主自助”。
2. 填写车辆：
   - 车辆编号：`CAR-DEMO-1`
   - 车主姓名：`Demo User`
   - 电池容量：80
   - 密码：`123456`
3. 点击“注册车辆”。
4. 点击“设置/更新密码”。
5. 选择快充，申请电量 30。
6. 点击“提交充电请求”。
7. 回到“调度控制台”，点击“执行调度”。
8. 回到“车主自助”，点击“刷新车辆状态”。
9. 查看“系统分配”和“当前服务进度”，确认车辆已进入某个充电桩队列。
10. 如果提示“可开始充电”，点击“启动充电”。
11. 实际电量填 30，点击“结束充电并生成账单”。
12. 点击“查询今日账单”，查看账单；单击账单行或“查看详单”查看分时计费详单。

## 演示三：优先级故障调度

1. 进入“调度控制台”。
2. 点击“清空数据”。
3. 点击“导入排队请求”。
4. 点击“执行调度”。
5. 进入“运营管理”。
6. 选择一个有排队车辆的快充桩，例如 `F-1`。
7. 点击“上报故障”。
8. 策略选择“优先保障中断车辆”。
9. 点击“确认上报并重调度”。
10. 查看队列状态，说明故障桩队列车辆优先迁移到其它同类型充电桩。

## 演示四：时间顺序故障调度

1. 回到“调度控制台”。
2. 点击“清空数据”。
3. 点击“导入排队请求”。
4. 点击“执行调度”。
5. 进入“运营管理”。
6. 对一个快充桩触发故障。
7. 策略选择“按请求时间重排”。
8. 点击“确认上报并重调度”。
9. 查看队列状态，说明系统按请求时间重新排列同类型未开始充电车辆。

## 演示五：老师测试用例

1. 进入“测试用例”。
2. 点击“运行用例”。
3. 查看“老师样例”表，确认 06:00、06:05、07:05、07:10、07:15 的示例值均为“通过”。
4. 查看“事件结果表”，说明系统按照老师表中的事件从 06:00 回放到 09:30。
5. 如需填表，点击“复制结果”，将表格内容粘贴到临时表格中再整理到老师给的 Excel。

## 可讲解要点

- 快充请求只进入快充桩，慢充请求只进入慢充桩。
- 普通调度目标是完成时间最短。
- 结束充电时系统按峰/平/谷分时电价生成账单和详单。
- 故障时暂停等候区叫号，先处理受影响车辆。
- “测试用例”页使用模拟时间，不会污染普通演示数据。
- H2 数据库降低本机验收部署复杂度。
